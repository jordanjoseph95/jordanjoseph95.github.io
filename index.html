Documentation

Technical write-ups from my AWS learning journey

Compute 3 docs
Amazon EKS — Part 1 Used eksctl to launch an EKS cluster on EC2; CloudFormation auto-provisioned the VPC, IAM roles, and node group; fixed IAM permission errors and tested Kubernetes self-healing by deleting nodes.
→
Amazon EKS — Part 2 Cloned a Flask app from GitHub, built a Docker image (fixing a Docker group permissions error along the way), and pushed it to Amazon ECR ready for deployment.
→
Amazon EKS — Part 3 Wrote Deployment and Service YAML manifests for the Flask app, applied them with kubectl, and explored how Kubernetes maintains desired replica state through self-healing.
→
Networking 9 docs
Virtual Private Cloud (VPC) Built a VPC from scratch with a custom IPv4 CIDR block, a public subnet, and an internet gateway; compared the AWS Console to CloudShell for creating the same resources.
→
VPC Connectivity Connected to EC2 instances via Instance Connect, fixed a missing SSH inbound rule, and confirmed instance-to-instance ping by adding an ICMP security group rule.
→
EC2 Networking Built a full VPC with public and private EC2 instances; configured route tables, security groups, and key pairs; then rebuilt everything in one step using the VPC wizard.
→
VPC Endpoints Created an S3 Gateway endpoint so an EC2 instance accesses S3 privately; applied a bucket policy restricting access to endpoint traffic only, and updated the VPC route table to use it.
→
Network Monitoring Enabled VPC Flow Logs to CloudWatch with a custom IAM role, peered two VPCs and fixed the route tables, then used Log Insights to query bytes transferred between instances.
→
VPC Peering Created a peering connection between two VPCs with non-overlapping CIDRs, updated both route tables, and confirmed cross-VPC ping after assigning an Elastic IP and fixing an ICMP rule.
→
Private Networking Created a private subnet alongside a public one, attached a dedicated route table with no internet route, and configured a custom network ACL to deny all inbound and outbound traffic.
→
S3 Networking Launched a VPC and EC2 instance, configured AWS credentials with aws configure, and used the CLI to list bucket contents and upload a file to S3.
→
Network Security Configured route tables, security groups, and network ACLs inside a VPC; contrasted default vs custom ACL behaviour; and used EC2 Global View to audit resources across multiple regions.
→
Security 1 doc
IAM — Identity & Access Management Wrote a JSON IAM policy scoping EC2 access to development-tagged instances, assigned it via a user group, and used the IAM Policy Simulator to confirm production actions were correctly denied.
→
Storage 1 doc
Host a Static Website on S3 Created an S3 bucket in eu-west-2, enabled static website hosting, fixed a 403 error by making files public via ACLs, and added a bucket policy to prevent the index file from being deleted.
